## Fixes

### add -p hunk edit: write edit file inside .git (fixes 19, 24, 28)
`run_editor_on_text` wrote the hunk-edit buffer to a `$TMPDIR` NamedTempFile.
Git writes it to `$GIT_DIR/addp-hunk-edit.diff` (via
`strbuf_edit_interactively(..., "addp-hunk-edit.diff", ...)`). t3701's fake editor
does a relative `mv -f patch "$1"`, which broke when `$1` lived in `$TMPDIR`
(cross-dir / perms). Switched to `git_dir.join("addp-hunk-edit.diff")`, threading
`&repo.git_dir` through `edit_hunk_and_apply`. Editor resolution order unchanged
(GIT_EDITOR/VISUAL/EDITOR).

### add -p hunk edit: garbage/headerless patch rejected (fixes 21)
The body parser required a `@@` header (`saw_body`) before processing lines, so a
totally garbled patch (no header) produced empty old/new and silently aborted.
Now anchors the body at the original hunk position when no header is present and
treats unrecognized lines as full-text context; a garbage patch then fails
`locate_hunk` and prints "patch does not apply" + "Your edited hunk does not
apply." (matching git). Bare-context lines (t28's trailing `c`) still work.
