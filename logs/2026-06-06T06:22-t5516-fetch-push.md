# t5516-fetch-push mop-up (ticket c87e2b)

Continuing from prior agent at 96/124.

## Lesson learned
Running `--run=<N>` in isolation is MISLEADING for t5516: many subtests depend
on the `setup` test populating `$the_first_commit`/`$the_commit`. With `--run=56`
alone, those vars are empty so `mk_test` issues a delete refspec and the test
follows a different path. ALWAYS validate with a full file run.

## Fix 1: push rejects empty branch name entries (subtests 56, 57)
Upstream `handle_config` in remote.c returns -1 for any `branch.<empty>.X`
(empty subsection), which the config machinery turns into a fatal
`bad config variable 'branch..X' in file ... at line N` while loading remote
config during push. grit was missing this; the no-arg push proceeded and the
remote rejected with `not a valid ref`.

Added `reject_empty_branch_subsection(&config)` in `grit/src/commands/push.rs`,
called right after config load in `run()`. It scans config entries for keys of
the form `branch..{remote,pushremote,merge}` (canonical key has an empty middle
component) and emits the upstream fatal error with file/line, exiting 128.

Result: 56 and 57 now pass. 98/124.

## Fix 2: push refmap for colon-less CLI refspecs (subtests 105, 107)
Mirrors Git `set_refspecs`/`refspec_append_mapped` (builtin/push.c). A colon-less
command-line ref (e.g. `main`) that uniquely names a local head is rewritten via:
1. `remote.<name>.push` as a refmap (test 105: `+refs/heads/*:refs/remotes/src/*`
   maps `main` -> `+refs/heads/main:refs/remotes/src/main`).
2. else if `push.default = upstream`, the branch's single `branch.<b>.merge` ref
   (test 107: `branch.main.merge refs/heads/trunk` -> `main:refs/heads/trunk`).

Added `map_cli_refspecs_via_refmap` + `query_push_refmap_dst` in push.rs and call
it in `run()` (made `args` mut) before computing `push_refspecs_from_config`.
Uses existing `count_refspec_match_push` (DWIM local-head match) and `match_glob`.

Result: 105, 107 pass; 106 still passes. 99/124.
