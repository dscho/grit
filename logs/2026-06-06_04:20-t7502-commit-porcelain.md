Ticket: bdf942

## Findings

- `commit --trailer` was parsed but not applied to the commit message. Wiring
  it through `grit_lib::interpret_trailers::process_trailers` fixed subtests
  10-24 and 26; commit mode must set `no_divider` so `---` in the body is not
  treated as a trailer divider.
- `-F` with `-e` bypassed the editor, and non-verbatim `-F` messages could keep
  stale raw bytes instead of the cleaned message.
- The commit path overwrote `.git/COMMIT_EDITMSG` with the cleaned commit body
  before checking for a `commit-msg` hook; when no hook exists, Git leaves the
  editor template available for status/comment assertions.
- `commit.status` config and explicit `--status` were ignored in the editor
  template path.
