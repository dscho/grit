## Fixes landed

1. **`%w(,1,2)` wrap spec (subtest 7)** ŌĆö `parse_wrap_spec` in `grit/src/commands/log.rs`
   required every comma-separated field to be non-empty digits, but Git's `pretty.c`
   case 'w' uses `strtoul`, so an empty leading segment yields 0 and parsing continues
   while the next byte is `,`. Rewrote with a `strtoul_prefix` helper that mirrors C
   `strtoul` (skip ws, consume digits, saturate). `%w(,1,2)` now -> (0,1,2).

2. **`-P`/`--perl-regexp` with no libpcre2 (subtest 41)** ŌĆö grit is not linked against
   libpcre2 (harness leaves `USE_LIBPCRE2` unset, so the PCRE prereq is off). Upstream
   Git dies "cannot use Perl-compatible regexes when not compiled with USE_LIBPCRE".
   Added that die in `run` when the effective grep pattern type is Perl and a
   grep/author/committer/grep-reflog pattern is present.

3. **`--graph` body rail prefix + inter-commit separator (subtest 59)** ŌĆö
   `write_graph_interleaved_commit_msg` only emitted a graph prefix for body lines while
   `!is_commit_finished()`; once the graph finished, trailing body lines (Author/blank/
   subject for `--pretty=short` merges) got no rail. Now mirror Git's `graph_show_strbuf`/
   `graph_show_oneline`: call `graph.next_line()` for every body line after the first ŌĆö
   it yields a *padding* line (the `| | ` rails) once the structural graph rows are done.
   Also restructured `run_graph_log` to emit the inter-commit blank as a railed separator
   line *between* commits (Git `show_log`: `shown_one && !use_terminator` ->
   `graph_show_padding`), gated on `log_wants_blank_line_between_commits`, and strip the
   single trailing blank that `format_commit` appends (so it isn't double-emitted nor
   railed after the final commit).
