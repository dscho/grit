## 2026-06-03 — `t3000-ls-files-others.sh`

- Reproduced current `t3000` as 8/15 after a fresh run.
- Root causes:
  - `ls-files --others` incorrectly suppressed the stdout redirection target whenever other
    untracked files existed. Git includes that already-created file in `--others` output.
  - `ls-files --others --directory` collapsed file-level pathspec matches such as
    `untracked/deep/path` and `untracked/*.c` to their parent directory, while Git preserves the
    individual file when no ancestor directory satisfies that pathspec.
  - `--directory --no-empty-directory` recursed into non-empty untracked directories but then kept
    direct child files instead of collapsing wholly untracked top-level directories back to
    `dir/`.
- Fix:
  - Removed redirection-target filtering.
  - Added pathspec-aware directory collapse that splits preserved file-level matches from
    collapsible directory matches.
  - Kept whole top-level untracked directories collapsed when there is no tracked content under
    that top-level path.

## 2026-06-03 — `t3200-branch.sh`

- Reproduced current `t3200` as 147/167.
- Fixed branch/worktree behavior:
  - Branch renames now update the main worktree `HEAD` as well as linked worktree `HEAD`s when the
    rename is run from a linked worktree.
  - Unborn branch renames across worktrees rewrite the relevant `HEAD` symrefs instead of
    rejecting the branch for having no commit yet.
- Fixed deletion safety:
  - `branch -d` checks a branch against its configured upstream/base when present, falling back to
    `HEAD` only when there is no upstream. This matches orphan-HEAD deletion cases and the
    tracked-base deletion tests.
- Fixed listing and parsing:
  - `--abbrev` no longer consumes a following branch pattern as its optional value; values must use
    `--abbrev=<n>`.
  - Detached branch descriptions prefer active rebase state over stale bisect state, producing
    `rebasing main` / `rebasing detached HEAD <oid>` rows during `branch --list`.
- Fixed config/ref edge cases:
  - Branch copy now inserts copied config sections next to each source section while preserving
    comments/formatting.
  - `rev-parse --verify -q` treats a self-referential symbolic ref as existing so `branch -d` can
    delete it and report the symbolic target.
- Fixed tracking setup:
  - `branch.autoSetupMerge=simple` only tracks matching branch names and only when the matching
    fetch refspec source is under `refs/heads/`.
  - Automatic local-branch tracking now detects ambiguous fetch refspecs and emits Git-compatible
    advice.
  - Explicit tracking validates the start point before writing the new branch, avoiding stale refs
    after failed `--track` attempts.
  - Explicit tracking of a remote-tracking branch now derives the merge ref from the matching fetch
    refspec source, including non-standard `refs/remotes/*:refs/remotes/*` mappings.
