## Root causes fixed (all in range_diff.rs)
1. `--output-indicator-new >` etc. were passed as a separate value token; grit's
   `log` arg splitter does not list those options as value-taking, so `>` was
   parsed as a revision (`fatal: ambiguous argument '>'`) and EVERY subtest after
   setup bailed. Switched to the inline `--output-indicator-new=>` form.
2. Inner patch-vs-patch diff was a full LCS dump (printed unchanged metadata /
   message and gave no `@@`-collapsed hunks). Rebuilt it as a real Myers unified
   diff (3 context lines), headers + line counts suppressed, with the range-diff
   `## name ##` / `@@ name` section-header funcname driver reproduced locally.
3. Commit-message body lines were de-indented; Git keeps the full 4-space-indented
   line (trailing ws stripped). Fixed.
4. Diff-section summaries only emitted `## new_name ##`; now the full git-diff
   header is parsed into new/deleted/rename/mode-change state to render
   `## x (new) ##`, `## x (deleted) ##`, `## a => b ##`, `## x (mode change …) ##`,
   including hunk-less (pure mode change) sections.
5. Patch-pairing cost used the full-LCS line count, hugely inflating it so changed
   commits never correlated. Now counts a real 3-context unified diff (Git's
   `diffsize`), matching Git's correspondence.
6. Added `--submodule[=<fmt>]` (accepted/ignored) and `--dual-color`; reordered
   argv so options may follow the range operands (clap `trailing_var_arg` was
   swallowing post-operand options like `--color --dual-color`).
7. Implemented Git `--dual-color` inner-diff rendering (reverse sign + bold/faint
   colors keyed on the inner line's first char, fraginfo `@@` headers, recolored
   `!`/`=` pair-header subject), mirroring `diff.c`'s `emit_diff_symbol` /
   `emit_line_0` exactly. Pair-header subject re-color (`reset`+`color_commit`)
   was also missing and is fixed.
