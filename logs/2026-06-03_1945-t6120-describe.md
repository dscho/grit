# t6120 describe

Baseline after syncing current `origin/main`: `t6120-describe.sh` was `54/103` in
`data/test-files.csv`.

Claimed `t6120-describe.sh` because it had the largest remaining t6 failing count.

Implemented this increment:

- Count describe distances as commits reachable from target but not the selected tag, instead of
  raw BFS parent depth.
- Keep tag/ref metadata through candidate selection so annotated tags, `--all`, and renamed tag
  refs can format differently.
- Add `--no-exact-match`, `--no-match`, `--no-exclude`, and `--no-long` parsing needed by the
  upstream describe tests.
- Apply `--match`/`--no-match` and `--exclude`/`--no-exclude` in argv order for describe.
- Make `git rev-parse <tag>-<n>-g<abbrev>` fall back to the unique commit abbreviation when the
  tag/count part is not a strict match, matching native Git behavior.
- Format exact annotated `describe --contains` results as `<tag>^0`.
- Warn and use the embedded annotated-tag name when a tag ref is renamed locally.
