# t0090-cache-tree (ticket fbeb9b)

## Symptom
Harness reported `status=error`, 0/0 — the script aborted before a `# Tests:`
summary. Log stopped after `ok 11`, then `FATAL: Unexpected exit with code 0`.

## Root cause
Not a grit crash. Test 12's `test_cache_tree` helper calls the recursive shell
function `generate_expected_cache_tree`, which runs
`git ls-tree --name-only -d HEAD -- "$pathspec/$subtree/"` to find sub-subtrees.

grit's `ls-tree -d HEAD -- dir/` (trailing slash) incorrectly emitted the
directory entry `dir` **itself** in addition to entries inside it. Real git's
`show_tree_common`/`show_recursive` (builtin/ls-tree.c) only prints an
intermediate tree node when `LS_SHOW_TREES` is set; for a plain `-d` with a
trailing-slash pathspec the directory entry itself is NOT a leaf and must be
suppressed (only entries strictly below it are shown).

Because grit kept re-emitting `dir`, the shell recursion `dir -> dir/ -> dir ->
...` never terminated; eventually a subshell `exit 0` propagated and tripped
test-lib's EXIT trap → FATAL.

Confirmed against real git:
- `ls-tree -d HEAD -- dir/`     -> `dir/sub`         (NOT `dir`)
- `ls-tree -d HEAD -- dir`      -> `dir`
- `ls-tree -r -d HEAD -- dir/`  -> `dir`, `dir/sub`  (-d -r auto-sets LS_SHOW_TREES)
- `ls-tree -t HEAD -- dir/`     -> `dir`, ...         (-t = LS_SHOW_TREES)

## Fix
grit/src/commands/ls_tree.rs, in `list_tree`'s recurse block: the
intermediate-node print was gated on `args.show_trees || args.only_trees`.
Dropped the `|| args.only_trees` so it only prints when `show_trees` is set
(which `-t` sets directly and `-d -r` auto-sets at line ~125), matching git's
`LS_SHOW_TREES` semantics.

## Result
0/0 (error) -> 21/22 passing.
