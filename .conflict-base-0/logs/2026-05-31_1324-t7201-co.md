## Changes

### grit/src/commands/checkout.rs
- Removed duplicate literal `error: ` prefix from the path-unmerged / conflict-style
  bail! strings so main.rs supplies the single prefix (27, 42).
- Peeled the merge tri-state flags (`-m`/`--merge`/`--no-merge`/`--conflict[=x]`/`--no-conflict`)
  out of `args.rest` so they are never mistaken for a revision or pathspec (38-41).
- Added a pre-scan over all matched paths for unmerged entries before writing any file in
  `checkout -- <paths>` so an unmerged path fails atomically with nothing written (27).
- Rewrote `unmerge_paths_in_index` to mirror git `unmerge_index`/`unmerge_index_entry`:
  restore stages from resolve-undo for resolved paths, leave already-unmerged live stages
  intact instead of deleting them (32-37).
- `merge_branch_working_tree` (`checkout -m`): build the final index from the destination
  tree at stage 0 (so `git diff --cached` is empty) while the work tree keeps the merged
  content (so `git diff` reports `M`); keep unmerged stages only for genuinely conflicting
  paths. Refresh cached stat only when work-tree content hashes to the index blob (5, 6).
- `switch_to_tree`: refresh the cached index stat only for entries whose work-tree content
  actually hashes to the index blob, so a carried-over locally-modified file (blob unchanged
  across the switch, never rewritten) keeps a stale stat and is re-hashed and reported `M`
  instead of being treated as clean by the diff stat fast-path (4).
- `check_dirty_worktree`: detect a staged deletion (path in HEAD, removed from index) that the
  target branch modifies and refuse the switch with "would be overwritten" (10).
- `checkout_conflicted_path_with_merge`: always run the configured custom merge driver for the
  leaf merge; `merge.<driver>.recursive=binary` only governs the virtual-ancestor merge, not the
  leaf, so it must not short-circuit to the ours blob (45).

### grit/src/commands/read_tree.rs
- Added `-q`/`--quiet` (OPT__QUIET): suppresses the "would be overwritten" feedback while still
  failing with exit 128 (used by `git merge`'s trial read-tree; t7201 10 second half).
- `two_way_merge`: when a path is absent from the index but present in both trees, conflict if the
  trees differ (target modifies a staged-deleted path) instead of resurrecting the file; keep the
  deletion when the trees agree. Gated on a non-empty (non-unborn) index so `read-tree -m H M` with
  an empty index still fast-forwards (preserves t1001 "no carry forward").

### grit/src/commands/merge_resolve.rs
- Set the new `quiet: false` field on the internal `read_tree::Args` initializer.

### grit-lib/src/merge_trees.rs
- Removed the `checkout_merge` special-case that forced 2-way conflict markers for every
  ours!=theirs path; the normal three-way line merge now records a single stage-0 entry for clean
  auto-merges and honors `presentation.style` for genuine conflicts (`--conflict=diff3`) (5, 6, 9).

### grit-lib/src/merge_file.rs
- Extended a changed region across contiguous changed base lines from either side so interleaved
  deletions (base `abcde`, ours `ace`, theirs `bd`) form one overlapping conflict region instead of
  alternating clean one-sided hunks that wrongly auto-resolved to empty (8). 14 merge_file unit
  tests still pass.
