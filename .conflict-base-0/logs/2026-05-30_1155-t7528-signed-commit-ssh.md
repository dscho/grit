# t7528-signed-commit-ssh — SSH commit signing & verification

Branch: `wf/p3/t7528-signed-commit-ssh`
Result: **green** — 26/26 real subtests pass + 3 `test_expect_failure` known
breakages (27/28/29, PGP double-signature TODOs that remain expected-to-fail).
Baseline was 4/29.

## What was wrong
SSH signing/verification (`gpg.format=ssh`, `ssh-keygen -Y`) was entirely
unimplemented in grit — `sign_buffer`/`verify_commit` short-circuited with
"ssh signing is not supported". The first subtest also exposed two unrelated
CLI/rebase bugs (merge `-S`, rebase `-f`, and a force-rebase cherry-pick base
bug) plus a missing feature (rebase did not sign replayed commits).

## Changes (all in this worktree)

### grit-lib/src/signing.rs (core SSH support)
- `GpgFormat::from_signature` — detect the verifier format from the signature
  armor (`get_format_by_sig`); verification now picks the program from the
  signature, not from `gpg.format`.
- `GpgFormat::from_name`/`name` reused; `GpgConfig` now carries per-format
  programs (`gpg.<fmt>.program` / `gpg.program`) plus `gpg.ssh.allowedSignersFile`
  and `gpg.ssh.revocationFile` (leading `~/` expanded).
- `sign_buffer_ssh` — `ssh-keygen -Y sign -n git -f <key> [-U] <buf>`, literal
  key handling (`key::` / `ssh-` via `is_literal_ssh_key`), reads `<buf>.sig`,
  strips CR, cleans up temp files.
- `verify_ssh_signed_buffer` — `find-principals` → `verify` per principal (or
  `check-novalidate` when none match → untrusted), `-Overify-time=` derived from
  the committer/tagger timestamp in local time (via `git_date::show::show_date`),
  honors `revocationFile`. Output assembled with a `stripspace` port so the
  `Good "git" signature ...` line stays separate from appended stderr.
- `parse_ssh_output` — sets result/trust/signer/key/fingerprint per
  gpg-interface.c (`G`+Fully for known principal, `G`+Undefined for unknown key,
  `B`/Never otherwise).
- `SignatureCheck::verifier_failed` — carries the ssh verify return code so an
  untrusted-but-`Good` signature still fails `verify-commit` (subtest 5/12).

### grit/src/commands/merge.rs
- `-S`/`--gpg-sign` now `require_equals` so a bare `-S` does not swallow the
  following positional (`git merge -S side`).

### grit/src/commands/rebase.rs
- Added `-f`/`--force-rebase` short option (was `--no-ff`/`--force-rebase`
  long-only) and `-S`/`--gpg-sign`/`--no-gpg-sign`.
- Cherry-pick merge base: use the picked commit's parent tree (not the onto
  tree) for clean dependent-chain replays under force-rewrite, so
  `rebase -f HEAD^^` over sequential same-file edits no longer spuriously
  conflicts. Gated on `force_rewrite_commits && head_tree == parent_tree` to
  avoid regressing the `--continue` conflict geometry (t3407 subtests 5/13).
- Persist a `gpg-sign-opt` rebase-state file (from `-S`/`commit.gpgsign`) and
  sign each replayed commit via `write_replayed_commit`, so force-rebased SSH
  commits (sixth-signed/seventh-signed) verify.

### grit/src/commands/pull.rs
- Pass the new rebase `gpg_sign`/`no_gpg_sign` fields when building rebase args.
