## Changes (all grit source, no test files touched)

1. grit/src/commands/submodule.rs
   - submodule_log_one(): join `--pretty=<fmt>` (was `--pretty` + fmt as two
     argv, which grit's log parser rejected → summary subprocess died →
     "Submodule changes to be committed" block dropped). Fixes 63, 64, 70.
   - submodule summary `--files`: iterate index gitlinks directly (like
     `git diff-files --ignore-submodules=dirty`) instead of requiring a
     `.gitmodules` entry; produces "Submodules changed but not updated".
   - submodule_ignore_all_for_summary / gitmodules_ignore_all_map: read
     `submodule.<name>.path/ignore` from `.gitmodules` url-independently and
     honor `.git/config` precedence, so ignore=all suppresses the summary for
     unregistered submodules. Fixes 93, 94.

2. grit/src/commands/diff_files.rs
   - collect_changes(): restore the `stat_agrees` guard so a stat-dirty entry
     whose content still hashes to the index OID is still reported as `M`
     (matches git run_diff_files, which never re-hashes to clear). Fixes 71.

3. grit-lib/src/diff.rs
   - New `refresh_index_stat_content_verified()`: content-verified index-stat
     refresh (only adopt worktree stat when content still matches the OID),
     mirroring git refresh_index/refresh_cache_ent.

4. grit/src/commands/status.rs
   - format_long: resolve effective submodule-ignore (CLI > .git/config >
     .gitmodules > diff.ignoreSubmodules), suppress/annotate gitlinks
     (`(new commits, modified content, untracked content)`), print the
     "commit or discard ... content in submodules" hint. Shared helper
     `submodule_display_decision`. Fixes 78-90, 92.
   - Multi-char `core.commentChar` used in full as the long-status comment
     leader (and bare string for the untracked/ignored column indent).
     Fixes 91.
   - Refresh the index via the shared helper before writing it. Fixes 56.

5. grit/src/commands/reset.rs
   - refresh_stage0_index_stats_from_worktree now delegates to the shared
     content-verified refresh (was unconditionally copying worktree stat into
     every stage-0 entry, falsely cleaning a modified file after `reset HEAD^`).
     This was the real blocker for 108.

6. grit/src/commands/stash.rs
   - do_push_staged / apply_stash_impl call the shared refresh before writing
     the index, keeping t3903 'stash refreshes the index' (27/28) passing
     alongside the diff-files stat-dirty change.

7. grit/src/commands/commit.rs
   - commit --dry-run honors `-u<mode>`/`--untracked-files=no` (prints
     "Untracked files not listed (use -u option to show untracked files)")
     and applies the shared per-submodule ignore decision. Fixes 108.
