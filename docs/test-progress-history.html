Grit harness test progress — fixed denominator

Passing tests as a share of the full suite (fixed denominator = 42,442 tests), per committed revision.

Every point is the recorded passing-test count divided by a single fixed full-suite total of 42,442 (today's suite size), so denominator changes no longer distort the curve. Green line = end-of-day value; faint blue dots = each commit. Apr 2–8 from docs/index.html, Apr 9 onward from docs/test-progress.svg. (Apr 1 is omitted: that day counted ported test files, not individual tests, so it isn't comparable.)

The cliffs on Apr 14 and Jun 1 are not denominator artifacts — fixing the denominator does not remove them. On those dates the harness re-baselined how it counts a passing test: across the same ~1,400 files the recorded passing count itself jumped (Apr 14: 37,628→18,625; Jun 1: 22,236→34,149). Those are measurement-methodology changes, not 19k regressions or 12k overnight fixes. Red dashed lines mark them. Source data: docs/test-progress-history.csv (date, passing, percent_of_full).
