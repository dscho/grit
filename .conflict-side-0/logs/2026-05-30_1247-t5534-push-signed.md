## Fix 1 — empty tree in pack-objects (subtests 2, 3, and unblocks 4/5/8/12)
`grit/src/commands/pack_objects.rs:read_object_from_repo` did not special-case
the well-known empty tree `4b825dc642cb6eb9a060e54bf8d69288fbee4904`. The
thin-pack reachability walk over `--allow-empty` commits therefore bailed with
`missing object in non-promisor repository`, breaking every push to a fresh
receiver (the `prepare_dst` helper). Added the same empty-tree guard that
`Odb::read` / `Odb::exists` already carry (canonical + legacy typo hash),
returning an empty `Tree` object. Verified manually:
`grit push dst main:noop main:ff main:noff` now exits 0 and creates all refs.

## Fix 2 — implement signed push / push certificates (subtests 4, 5, 8, 12)
`--signed` was parsed but ignored ("accepted but not implemented"). Implemented
the push-certificate flow for the native/local transport:

- New `grit-lib/src/push_cert.rs`:
  - `prepare_push_cert_nonce` — RFC 2104 HMAC-SHA1 nonce `<stamp>-<hex>` keyed by
    `receive.certNonceSeed` over `<git-dir>:<stamp>` (port of
    `receive-pack.c:prepare_push_cert_nonce` / `hmac_hash`).
  - `build_push_cert_payload` — assembles the `certificate version 0.1` payload
    (`send-pack.c:generate_push_cert`).
  - `cert_env_from_check` / `PushCertEnv` — derives the `GIT_PUSH_CERT*` hook
    environment (`receive-pack.c`).
  - `verify_push_cert` — reuses `signing::verify_tag` (cert = payload + appended
    detached signature, same layout as a signed tag).
  - Unit tests for nonce shape/stability and payload contents.
- `grit/src/commands/push.rs`:
  - `PushCertMode` (Never / IfAsked / Always) parsed from `--signed[=...]` /
    `--no-signed`.
  - `prepare_signed_push_cert`: with no `receive.certNonceSeed`, `--signed`
    (Always) fails with `the receiving end does not support --signed push`
    (4/5); IfAsked is a no-op. With a seed, the cert is generated, signed via
    `signing::sign_buffer` (gpg/ssh), stored as a blob on the receiver, verified
    with the receiver's gpg config, and the `GIT_PUSH_CERT*` vars are appended to
    the receive-hook environment (8/12 GPGSSH).
  - Signing failure (e.g. empty `user.signingkey`, subtest 12 first leg)
    propagates as a push failure, matching Git.
